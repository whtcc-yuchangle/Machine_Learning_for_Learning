## 内容结构

### 第一章：朴素贝叶斯算法原理

#### 1.1 贝叶斯定理

朴素贝叶斯的核心是**贝叶斯定理**：

$$P(C|X) = \frac{P(X|C) \cdot P(C)}{P(X)}$$

其中：

| 符号 | 含义 |
|:---|:---|
| $P(C\|X)$ | **后验概率**：在已知特征 $X$ 的条件下，样本属于类别 $C$ 的概率 |
| $P(X\|C)$ | **似然（Likelihood）**：在类别 $C$ 下观察到特征 $X$ 的概率 |
| $P(C)$ | **先验概率**：类别 $C$ 在数据集中出现的概率 |
| $P(X)$ | **证据（Evidence）**：特征 $X$ 出现的概率（归一化因子） |

#### 1.2 "朴素"从何而来

朴素贝叶斯的"**朴素**"（Naive）指的是**特征条件独立假设**——假设所有特征之间相互独立：

$$P(X|C) = P(x_1|C) \cdot P(x_2|C) \cdot ... \cdot P(x_n|C) = \prod_{i=1}^{n}P(x_i|C)$$

这个假设在现实中往往不成立（例如手写数字中相邻像素高度相关），但朴素贝叶斯在实践中仍然表现良好。

#### 1.3 分类决策

对于给定的样本 $X$，朴素贝叶斯选择使后验概率最大的类别：

$$\hat{y} = \arg\max_{C} P(C|X) = \arg\max_{C} P(C) \prod_{i=1}^{n}P(x_i|C)$$

> 由于 $P(X)$ 对所有类别相同，分类时只需比较分子部分。

#### 1.4 三种常见变体

| 变体 | sklearn 类 | 适用场景 | 特征假设 |
|:---|:---|:---|:---|
| **高斯朴素贝叶斯** | `GaussianNB` | 连续特征 | 特征服从高斯（正态）分布 |
| **多项式朴素贝叶斯** | `MultinomialNB` | 文本分类（词频） | 特征为离散计数 |
| **伯努利朴素贝叶斯** | `BernoulliNB` | 二值特征 | 特征为 0/1（是否出现） |
| **补充朴素贝叶斯** | `ComplementNB` | 不平衡文本分类 | MNB 的改进版 |

本教程使用 **高斯朴素贝叶斯**，因为手写数字的像素值是连续数据。

#### 1.5 高斯朴素贝叶斯的数学原理

对于连续特征，假设每个特征在每个类别下服从**高斯分布（正态分布）**：

$$P(x_i|C) = \frac{1}{\sqrt{2\pi\sigma_C^2}} \exp\left(-\frac{(x_i - \mu_C)^2}{2\sigma_C^2}\right)$$

其中 $\mu_C$ 和 $\sigma_C^2$ 是从训练数据中估计的均值和方差。

> 高斯朴素贝叶斯无需超参数调优——模型通过计算每个类别下每个特征的均值和方差完成"训练"。

本章对应 notebook 中的实际代码单元格，逐步演示如何使用高斯朴素贝叶斯对手写数字进行分类。

#### 2.1 数据集介绍

使用 sklearn 自带的 **Digits Dataset**（手写数字数据集）：

| 属性 | 说明 |
|:---|:---|
| 样本数量 | 1,797 个 |
| 特征数量 | 64 个（8×8 像素灰度值，范围 0~16） |

### 第三章：模型评估

#### 3.1 评估指标

基于混淆矩阵可以计算多种评估指标：

```python
from sklearn.metrics import classification_report, accuracy_score, confusion_matrix

Y_pred = gnb.predict(Xtest)

# 分类报告（精确率、召回率、F1）
print(classification_report(Ytest, Y_pred, target_names=[str(i) for i in range(10)]))

# 整体准确率
print(f"准确率: {accuracy_score(Ytest, Y_pred):.4f}")
```

| 指标 | 公式 | 说明 |
|:---|:---|:---|
| **准确率** | (TP + TN) / Total | 所有预测正确的比例 |
| **精确率** | TP / (TP + FP) | 预测为正类中真正类的比例 |
| **召回率** | TP / (TP + FN) | 真正类中被正确找出的比例 |
| **F1 分数** | 2 × P × R / (P + R) | 精确率和召回率的调和平均 |

### 第四章：朴素贝叶斯算法特点

#### 4.1 优点

| 优点 | 说明 |
|:---|:---|
| **训练速度快** | 只需计算均值和方差，无需迭代优化 |
| **小数据集有效** | 在样本量少时也能取得不错效果 |
| **可解释性强** | 基于概率，输出有明确的概率含义 |
| **天然多分类** | 无需 One-vs-Rest 等策略 |
| **对缺失值不敏感** | 可忽略缺失特征进行计算 |
| **无需调参** | 高斯朴素贝叶斯没有需要调节的超参数 |

#### 4.2 缺点

| 缺点 | 说明 |
|:---|:---|
| **特征独立假设过强** | "朴素"假设现实中往往不成立 |
| **对特征分布敏感** | 若特征不符合高斯分布，效果会下降 |
| **零概率问题** | 若某特征值在训练集中未出现，概率为零（可用拉普拉斯平滑解决） |
| **连续特征需合理分布** | 不同变体对特征类型有严格要求 |

#### 4.3 适用场景

- **文本分类/垃圾邮件过滤**（配合 MultinomialNB）
- **实时预测**（训练快、预测快）
- **基线模型**（作为复杂模型的对比基准）
- **特征维度高但样本少的场景**
- **需要概率输出的场景**（如风险评分）

### 第五章：与其他算法对比

#### 5.1 朴素贝叶斯对比表

| 对比维度 | 朴素贝叶斯 | KNN | 决策树 |
|:---|:---|:---|:---|
| 核心思想 | 贝叶斯定理 + 条件独立 | 距离度量 + 投票 | 递归特征分裂 |
| 训练过程 | 计算统计量（极快） | 无（lazy learning） | 构建树结构 |
| 预测速度 | 快 | 慢（需计算所有距离） | 快（O(log n)） |
| 可解释性 | 中（概率输出） | 低 | 高（可视化） |
| 超参数 | 几乎无 | K 值 | 深度、分裂准则等 |
| 特征缩放 | 不需要 | 需要 | 不需要 |
| 非线性 | 天然支持 | 天然支持 | 天然支持 |
| 大数据集 | 好 | 差（预测慢） | 中（容易过拟合） |

#### 5.2 GaussianNB vs 其他贝叶斯变体

| 变体 | 特征类型 | 典型应用 |
|:---|:---|:---|
| **GaussianNB** | 连续值 | 图像分类、传感器数据 |
| **MultinomialNB** | 离散计数 | 文本分类、文档主题分类 |
| **BernoulliNB** | 二值 (0/1) | 垃圾邮件检测、关键词出现与否 |
| **ComplementNB** | 离散计数 | 不平衡文本分类（改进版 MNB） |

## 常见问题

### Q1: 为什么叫"朴素"贝叶斯？

因为它做了一个非常"天真"的假设：所有特征相互独立。这在实际问题中几乎不可能成立，但有趣的是，即使假设被违反，朴素贝叶斯仍然常常表现良好——尤其是在文本分类领域。

### Q2: GaussianNB 有超参数需要调吗？

```python
gnb = GaussianNB(
    priors=None,            # 先验概率，默认从数据中学习
    var_smoothing=1e-9      # 方差平滑项（防止除零）
)
```

- `priors`：可手动指定类别先验概率，不指定则从训练数据中计算
- `var_smoothing`：方差平滑项，默认 `1e-9`，几乎不需要调整

### Q3: 零概率问题是什么？怎么解决？

如果某个特征值在训练集的某个类别中从未出现，计算出的 $P(x_i|C)=0$，导致整个后验概率归零。

**解决方法**——拉普拉斯平滑：

$$P(x_i|C) = \frac{N_{x_i, C} + \alpha}{N_C + \alpha \cdot n}$$

其中 $\alpha$ 是平滑参数（通常取 1）。在 sklearn 中，MultinomialNB 和 BernoulliNB 默认 `alpha=1.0`。

### Q4: Digits 数据集 85.93% 的准确率算好吗？

| 模型 | 大致准确率 |
|:---|:---|
| 随机猜测（10 分类） | ~10% |
| 朴素贝叶斯（GaussianNB） | ~86% |
| KNN（k=5） | ~98% |
| SVM | ~99% |
| 简单神经网络 | ~98%+ |

朴素贝叶斯比随机猜测好得多，但比 KNN 和 SVM 低——因为像素值并不完全满足高斯分布假设，且特征之间的强相关性违反了"朴素"假设。但对于一个**无调参、训练瞬间完成**的模型来说，85.93% 已经相当不错。
